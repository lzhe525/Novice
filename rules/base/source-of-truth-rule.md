# source-of-truth-rule

## 适用主体

在 `{projectRoot}` 中依据文档与源码做决策或修改代码的 Agent。

## 规则陈述

**源码与真实配置文件是事实源（source of truth）。** `.ai/` 下的说明性文档、索引、注释类 Markdown 均为辅助上下文。

## 冲突时的处理

- 若 `.ai/` 中文档描述与 `{projectRoot}` 内实际源码或配置不一致，**以源码与配置为准**。
- Agent 不得仅依据过期的 `.ai/` 文档修改代码；发现冲突时应在回复中说明，并在有能力时按项目流程更新文档（后续 Skill 定义；MVP 阶段至少口头标记「文档疑似过期」）。

## 禁止事项

- 在未打开目标源文件的情况下，仅凭 `.ai/` 概述对生产代码做大范围改写。

## 与其它 Rule 的关系

- 与 `read-source-before-edit-rule` 配套：先读真实源文件，再编辑。
