# human-readable-doc-rule

## 适用主体

编写 `.ai/docs/**` 文档的 Agent 与审阅文档的人类。

## 规则陈述

文档在服务于 Agent 检索的同时，必须保持**人类可读**：语句完整、指代清晰、关键结论有依据说明（指向已读源文件路径，而非臆测）。

## 写作要求

1. 首段用两三句话说明文档范围与生成方式（例如「由 `scan-project-by-ai` 基于只读源码生成，非事实源」）。
2. 技术名词与代码标识符保持英文或与源码一致；解释性句子使用中文（与 `language-policy.md` 一致）。
3. 对不确定内容使用明确措辞：「待确认」「基于有限文件推断」，不得写为确定事实。

## 允许路径

- `{projectRoot}/.ai/docs/**`

## 禁止事项

- 虚构提交历史、部署环境或未读文件中的 API 行为。
- 将内部代号或缩写不加解释直接使用（至少首次出现处解释）。

## 与其它 Rule 的关系

- 落实 `source-of-truth-rule`：人类与 Agent 均应能根据文档找到对应源码核对。
