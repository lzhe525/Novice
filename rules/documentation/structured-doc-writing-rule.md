# structured-doc-writing-rule

## 适用主体

编写 `.ai/docs/**` 下项目文档的 Agent。

## 规则陈述

扫描与人工协作产生的文档必须**结构化**，优先使用：分级标题、表格、列表、固定字段小节；避免无标题的长段落堆砌。

## 推荐结构（项目概述类）

1. `#` 或 `##` 级标题区分：概述、技术栈、目录布局、构建与运行、已知风险、待确认项。
2. 「待确认项」须单独成节，列出需人类回答的问题编号。
3. 涉及路径与文件名时单独使用行内代码或表格列，保持与仓库一致。

## 允许路径

- `{projectRoot}/.ai/docs/**`

## 禁止事项

- 用散文替代应存在的表格（例如多版本依赖矩阵）。
- 省略文档类型所需的关键小节（见各 Scan Skill 的完成标准）。

## 与其它 Rule 的关系

- 与 `human-readable-doc-rule` 互补：结构化不等于堆砌术语。
- 与 `frontmatter-format-rule` 互补：front matter 字段名保持稳定、可枚举。
