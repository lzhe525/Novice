## 第二节：硬约束、风险与目录策略（对应 checklist 步骤 9）

### Q2.1 硬约束（必须遵守）

- **问题**：有哪些不可违反的约束（合规、API 版本、禁止改动的接口等）？
- **推荐回复格式**：
  - 硬约束：
    1. …
    2. …

### Q2.2 高风险区域

- **问题**：哪些路径或模块改动风险高（鉴权、支付、并发核心等）？
- **推荐回复格式**：
  - 高风险（相对 `{projectRoot}` 的路径或模块名）：`…`

### Q2.3 源码根目录

- **问题**：主要业务源码根路径（相对仓库根）？
- **推荐回复格式**：`例如 src/` 或 `packages/foo/src/`

### Q2.4 忽略目录（扫盘或 Agent 应跳过）

- **问题**：哪些目录应忽略（`node_modules`、`dist`、生成物等）？
- **推荐回复格式**：列表，每行一个相对路径

### Q2.5 只读或禁止自动修改的文件

- **问题**：哪些文件仅人类可改或禁止 Agent 修改？
- **推荐回复格式**：列表，每行一个相对路径或 glob

### Q2.6 语言策略

- **问题**：文档与回复语言偏好是否已在 `.ai/config/language-policy.md` 中体现并由人类确认？
- **推荐回复格式**：
  - 状态：`已打开 language-policy.md 并按指引完成 reviewedByHuman: true` / `尚未，需要协助阅读模板`
